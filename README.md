# 1. 요구분석

## 1.1 프로젝트 소개

일정 공유 웹 서비스는 여러 사용자가 자신의 일정을 등록하고 그룹 단위로 공유할 수 있는 웹 기반 일정 관리 시스템이다. 사용자는 그룹을 생성하거나 참여하여 팀 단위로 일정을 관리할 수 있으며, 그룹 캘린더를 통해 모든 구성원의 일정을 확인할 수 있다.

---

## 1.2 프로젝트 목적

기존 캘린더 서비스는 개인 일정 관리 중심인 경우가 많아 팀 단위 일정 조율이 어렵다. 본 프로젝트의 목적은 그룹 단위 일정 공유 기능을 제공하여 팀, 동아리, 스터디 등 다양한 협업 환경에서 효율적인 일정 관리를 가능하게 하는 것이다.

---

## 1.3 기능 명세

### 회원 기능

* 회원가입
* 로그인

### 그룹 기능

* 그룹 생성
* 그룹 참여
* 그룹 목록 조회

### 일정 기능

* 일정 등록
* 일정 조회
* 일정 수정
* 일정 삭제

### 확장 기능

* 일정 알림 기능
* 일정 충돌 검사
* 실시간 일정 업데이트

# 2. 설계

## 2.1 시스템 아키텍처

시스템은 다음과 같은 구조로 구성된다.

* Frontend : 사용자 인터페이스 제공
* Backend : 비즈니스 로직 및 API 처리
* Database : 사용자 및 일정 데이터 저장

### 시스템 구조

```
User
 │
 ▼
Frontend (React)
 │
 ▼
Backend (Node.js / Express)
 │
 ▼
Database (MongoDB 또는 MySQL)
```

---

## 2.2 데이터베이스 설계 (ERD)

### User

| 필드         | 설명     |
| ---------- | ------ |
| user_id    | 사용자 ID |
| email      | 이메일    |
| password   | 비밀번호   |
| name       | 사용자 이름 |
| created_at | 생성 날짜  |

---

### Group

| 필드         | 설명     |
| ---------- | ------ |
| group_id   | 그룹 ID  |
| group_name | 그룹 이름  |
| creator_id | 생성자 ID |
| created_at | 생성 날짜  |

---

### GroupMember

| 필드       | 설명     |
| -------- | ------ |
| id       | 고유 ID  |
| group_id | 그룹 ID  |
| user_id  | 사용자 ID |
| role     | 사용자 역할 |

설명
사용자와 그룹 사이의 다대다(N:M) 관계를 관리하는 테이블이다.

---

### Schedule

| 필드          | 설명     |
| ----------- | ------ |
| schedule_id | 일정 ID  |
| group_id    | 그룹 ID  |
| creator_id  | 작성자 ID |
| title       | 일정 제목  |
| description | 일정 설명  |
| start_time  | 시작 시간  |
| end_time    | 종료 시간  |
| created_at  | 생성 시간  |

---

## 2.3 API 설계

### 회원가입

POST /api/auth/register

요청

```json
{
  "email": "user@email.com",
  "password": "1234",
  "name": "홍길동"
}
```

응답

```json
{
  "message": "회원가입 성공"
}
```

---

### 로그인

POST /api/auth/login

요청

```json
{
  "email": "user@email.com",
  "password": "1234"
}
```

응답

```json
{
  "token": "JWT_TOKEN"
}
```

---

### 그룹 생성

POST /api/groups

요청

```json
{
  "group_name": "프로젝트 팀"
}
```

---

### 일정 등록

POST /api/schedules

요청

```json
{
  "group_id": 1,
  "title": "회의",
  "description": "프로젝트 회의",
  "start_time": "2026-04-01 14:00",
  "end_time": "2026-04-01 15:00"
}
```
