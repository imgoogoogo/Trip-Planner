### 프로젝트 소개

여행 계획을 효율적으로 관리하기 위해 지도 기반으로 여행 장소를 등록하고 여행 일정을 공유할 수 있는 웹 서비스를 개발한다.

사용자는 지도 위에서 여행 장소를 직접 선택하여 등록할 수 있으며, 등록된 장소를 기반으로 여행 일정을 구성할 수 있다. 또한 여행 일정을 다른 사용자와 공유하여 친구나 가족과 함께 여행 계획을 협업 방식으로 관리할 수 있다. 이를 통해 여행 장소와 일정 정보를 하나의 플랫폼에서 통합적으로 관리할 수 있도록 한다.

---

### 프로젝트 목적

여행을 계획할 때 여행 장소 정보, 일정, 지도 정보 등을 여러 애플리케이션에서 각각 관리하는 경우가 많아 정보 관리가 비효율적이며 전체 여행 동선을 직관적으로 파악하기 어렵다.

본 프로젝트의 목적은 지도 기반 인터페이스를 활용하여 여행 장소를 시각적으로 관리하고 여행 일정을 효율적으로 구성할 수 있는 시스템을 제공하는 것이다. 또한 일정 공유 기능을 통해 여러 사용자가 하나의 여행 계획을 함께 관리할 수 있도록 하여 여행 준비 과정을 보다 체계적으로 지원하는 것을 목표로 한다.

---

### 문제 정의

- 여행 장소 정보가 지도 앱, 메모 앱, 일정 앱 등 여러 애플리케이션에 분산되어 관리된다.
- 여행 일정과 장소 정보를 하나의 시스템에서 통합적으로 관리하기 어렵다.
- 지도 기반으로 여행 동선을 직관적으로 확인하기 어렵다.
- 여러 사람이 함께 여행을 준비할 때 일정 공유와 협업이 불편하다.

---

### 예상 사용자

- 개인 여행 사용자
  - 혼자 여행을 계획하며 여행 장소와 일정을 체계적으로 관리하고 싶은 사용자

- 그룹 여행 사용자
  - 친구, 가족, 동료 등 여러 사람이 함께 여행을 준비하며 여행 일정과 장소 정보를 공유하고 협업이 필요한 사용자

## 데이터베이스 설계 (ERD)

<img src="https://github.com/user-attachments/assets/4d625b80-9fac-4783-95b7-d1c3a546f57b" width="480" align="left" style="margin-right: 20px;">

### ERD 관계 설명
#### ① 한 사용자는 여러 여행에 참여할 수 있다.
#### ② 한 사용자는 여러 여행을 생성할 수 있다.
#### ③ 일정은 누가 작성했는지 기록한다. 
#### ④ 하나의 여행에는 여러 일정이 존재한다. 
#### ⑤ 하나의 장소는 여러 일정에서 사용될 수 있다. 
#### ⑥ 하나의 여행에는 여러 사용자가 참여할 수 있다.
#### ⑦ 하나의 여행에는 여러 장소가 등록될 수 있다.

## 데이터베이스 명세서

### 1. Users 테이블

사용자 계정 정보를 저장하는 테이블이다.

| 컬럼명        | 데이터 타입       | 제약조건                      | 설명        |
| ---------- | ------------ | ------------------------- | --------- |
| user_id    | INT          | PK, AUTO_INCREMENT        | 사용자 고유 ID |
| email      | VARCHAR(100) | UNIQUE, NOT NULL          | 사용자 이메일   |
| password   | VARCHAR(255) | NOT NULL                  | 사용자 비밀번호  |
| name       | VARCHAR(50)  | NOT NULL                  | 사용자 이름    |
| created_at | TIMESTAMP    | DEFAULT CURRENT_TIMESTAMP | 계정 생성 시간  |

### 2. Trips 테이블

사용자가 생성한 여행 계획 정보를 저장하는 테이블이다.

| 컬럼명         | 데이터 타입       | 제약조건                      | 설명     |
| ----------- | ------------ | ------------------------- | ------ |
| trip_id     | INT          | PK, AUTO_INCREMENT        | 여행 ID  |
| creator_id  | INT          | FK                        | 여행 생성자 |
| title       | VARCHAR(100) | NOT NULL                  | 여행 제목  |
| description | TEXT         |                           | 여행 설명  |
| start_date  | DATE         |                           | 여행 시작일 |
| end_date    | DATE         |                           | 여행 종료일 |
| created_at  | TIMESTAMP    | DEFAULT CURRENT_TIMESTAMP | 생성 시간  |

### 3. Trip_Members 테이블

여행에 참여하는 사용자 정보를 관리하는 테이블이다.

| 컬럼명       | 데이터 타입                 | 제약조건               | 설명     |
| --------- | ---------------------- | ------------------ | ------ |
| member_id | INT                    | PK, AUTO_INCREMENT | 참여자 ID |
| trip_id   | INT                    | FK                 | 여행 ID  |
| user_id   | INT                    | FK                 | 사용자 ID |
| role      | ENUM('owner','member') | DEFAULT 'member'   | 사용자 역할 |

### 4. Places 테이블

여행 중 방문할 장소 정보를 저장하는 테이블이다.

| 컬럼명         | 데이터 타입        | 제약조건               | 설명    |
| ----------- | ------------- | ------------------ | ----- |
| place_id    | INT           | PK, AUTO_INCREMENT | 장소 ID |
| trip_id     | INT           | FK                 | 여행 ID |
| name        | VARCHAR(100)  | NOT NULL           | 장소 이름 |
| latitude    | DECIMAL(10,7) | NOT NULL           | 위도    |
| longitude   | DECIMAL(10,7) | NOT NULL           | 경도    |
| description | TEXT          |                    | 장소 설명 |

### 5. Schedules 테이블

여행 일정 정보를 저장하는 테이블이다.

| 컬럼명         | 데이터 타입                          | 제약조건                      | 설명              |
| ----------- | ------------------------------- | ------------------------- | --------------- |
| schedule_id | INT                             | PK, AUTO_INCREMENT        | 일정 ID           |
| trip_id     | INT                             | FK                        | 여행 ID           |
| place_id    | INT                             | FK                        | 장소 ID (NULL 허용) |
| creator_id  | INT                             | FK                        | 일정 작성자          |
| title       | VARCHAR(100)                    | NOT NULL                  | 일정 제목           |
| description | TEXT                            |                           | 일정 설명           |
| start_time  | DATETIME                        |                           | 일정 시작 시간        |
| end_time    | DATETIME                        |                           | 일정 종료 시간        |
| visibility  | ENUM('public','trip','private') | DEFAULT 'trip'            | 일정 공개 범위        |
| created_at  | TIMESTAMP                       | DEFAULT CURRENT_TIMESTAMP | 생성 시간           |

## 2.3 API 설계

### 회원가입

POST /api/auth/register

요청

```json
{
  "email": "user@email.com",
  "password": "1234",
  "name": "홍길동"
}
```

응답

```json
{
  "message": "회원가입 성공"
}
```

---

### 로그인

POST /api/auth/login

요청

```json
{
  "email": "user@email.com",
  "password": "1234"
}
```

응답

```json
{
  "token": "JWT_TOKEN"
}
```

---

### 그룹 생성

POST /api/groups

요청

```json
{
  "group_name": "프로젝트 팀"
}
```

---

### 일정 등록

POST /api/schedules

요청

```json
{
  "group_id": 1,
  "title": "회의",
  "description": "프로젝트 회의",
  "start_time": "2026-04-01 14:00",
  "end_time": "2026-04-01 15:00"
}
```
