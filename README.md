### 프로젝트 소개

여행 계획을 효율적으로 관리하기 위해 지도 기반으로 여행 장소를 등록하고 여행 일정을 공유할 수 있는 웹 서비스를 개발한다.

사용자는 지도 위에서 여행 장소를 직접 선택하여 등록할 수 있으며, 등록된 장소를 기반으로 여행 일정을 구성할 수 있다. 또한 여행 일정을 다른 사용자와 공유하여 친구나 가족과 함께 여행 계획을 협업 방식으로 관리할 수 있다. 이를 통해 여행 장소와 일정 정보를 하나의 플랫폼에서 통합적으로 관리할 수 있도록 한다.

---

### 프로젝트 목적

여행을 계획할 때 여행 장소 정보, 일정, 지도 정보 등을 여러 애플리케이션에서 각각 관리하는 경우가 많아 정보 관리가 비효율적이며 전체 여행 동선을 직관적으로 파악하기 어렵다.

본 프로젝트의 목적은 지도 기반 인터페이스를 활용하여 여행 장소를 시각적으로 관리하고 여행 일정을 효율적으로 구성할 수 있는 시스템을 제공하는 것이다. 또한 일정 공유 기능을 통해 여러 사용자가 하나의 여행 계획을 함께 관리할 수 있도록 하여 여행 준비 과정을 보다 체계적으로 지원하는 것을 목표로 한다.

---

### 문제 정의

- 여행 장소 정보가 지도 앱, 메모 앱, 일정 앱 등 여러 애플리케이션에 분산되어 관리된다.
- 여행 일정과 장소 정보를 하나의 시스템에서 통합적으로 관리하기 어렵다.
- 지도 기반으로 여행 동선을 직관적으로 확인하기 어렵다.
- 여러 사람이 함께 여행을 준비할 때 일정 공유와 협업이 불편하다.

---

### 예상 사용자

- 개인 여행 사용자
  - 혼자 여행을 계획하며 여행 장소와 일정을 체계적으로 관리하고 싶은 사용자

- 그룹 여행 사용자
  - 친구, 가족, 동료 등 여러 사람이 함께 여행을 준비하며 여행 일정과 장소 정보를 공유하고 협업이 필요한 사용자

# 2. 설계

## 2.1 시스템 아키텍처

시스템은 다음과 같은 구조로 구성된다.

* Frontend : 사용자 인터페이스 제공
* Backend : 비즈니스 로직 및 API 처리
* Database : 사용자 및 일정 데이터 저장


## 2.3 API 설계

### 회원가입

POST /api/auth/register

요청

```json
{
  "email": "user@email.com",
  "password": "1234",
  "name": "홍길동"
}
```

응답

```json
{
  "message": "회원가입 성공"
}
```

---

### 로그인

POST /api/auth/login

요청

```json
{
  "email": "user@email.com",
  "password": "1234"
}
```

응답

```json
{
  "token": "JWT_TOKEN"
}
```

---

### 그룹 생성

POST /api/groups

요청

```json
{
  "group_name": "프로젝트 팀"
}
```

---

### 일정 등록

POST /api/schedules

요청

```json
{
  "group_id": 1,
  "title": "회의",
  "description": "프로젝트 회의",
  "start_time": "2026-04-01 14:00",
  "end_time": "2026-04-01 15:00"
}
```
